## 三、⚠️ 循环依赖 (6 组)

### 3.1 编译器核心循环 (9 模块) — **架构级**
```
前端入口 ↔ 增量缓存 ↔ 发射器 ↔ 常量 ↔ 编码器
  ↔ 硬件库 ↔ 热更新 ↔ 调试器 ↔ 自举兼容桥
```
**说明**: 编译器与运行时互相引用, 常量与发射器互相引用.  
**影响**: 不能独立提取任一子模块使用.  
**建议**: 抽取**纯常量层** (无依赖), 将循环改为 DAG.

### 3.2 GPU 硬件循环 (13 模块)
```
AMD/NVIDIA 多个库 ↔ 内存/显存 ↔ FFI 桥接 ↔ 内核编译器
```
**说明**: GPU 生态抽象层互调.  
**建议**: 抽取 `GPU/接口层.nova` 纯接口定义, 具体实现 依赖接口.

### 3.3 跨语言循环 (8 模块)
```
Java/Cpp/Python/C/Rust/Go 前端 ↔ 统一语义图 ↔ 模块重建器
```
**说明**: 多前端共享统一语义图.  
**建议**: `统一语义图` 应是**叶节点** (只被依赖, 不依赖具体前端).

### 3.4 裸机循环 (5 模块)
```
无GC运行时 ↔ 原子操作编码 ↔ 特权指令编码 ↔ 易变访问 ↔ x86_64代码生成器
```
**建议**: `代码生成器` 不应依赖裸机扩展, 应由裸机扩展调用代码生成.

### 3.5 Forth 转换器循环 (2 模块)
```
Forth转换器 ↔ 转换验证
```
小范围, 可合并.
