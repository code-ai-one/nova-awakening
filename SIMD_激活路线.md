## 二、激活路线 · 三档难度

### L1 (低难度) · 显式 intrinsic 函数 · 1-2 周
**思路**: 引入用户调用的内置函数

```nova
__simd_浮点加向量(数组a, 数组b, 结果, 元素数)
__simd_浮点乘向量(数组a, 数组b, 结果, 元素数)
__simd_点积(数组a, 数组b, 元素数)  // 返回 f64
```

**实现**:
1. 在代码生成器里识别特殊函数名, 发射 SIMD 循环字节码
2. 循环: `load → addpd/mulpd → store`, 每迭代处理 2 个 f64
3. 尾部: 标量处理 N%2 个元素

**收益**: AI 算子 / 矩阵运算**用户显式调用**可获 2x 加速
**风险**: 低 (只加新符号, 不改主路径)

### L2 (中难度) · 模式识别循环向量化 · 1-2 月
**思路**: 编译器自动识别 for 循环模式

```nova
当 (j < N) {
    结果[j] = a[j] + b[j]
    j = j + 1
}
// → 自动检测 + SIMD 化
```

**需要**:
1. 循环形态分析 (归纳变量 j, 步长 1, 边界 N)
2. 依赖分析 (无跨迭代读写冲突)
3. 类型确认 (a, b 是 `__浮点列表__`)
4. 内存布局保证 (连续 f64)
5. 对齐/不对齐分支
6. 尾部 peel
7. 寄存器分配集成 xmm 族

**收益**: matvec / axpy / 卷积等常见循环自动 2-4x
**风险**: 中 (依赖分析难, 错误会产生错误结果)

### L3 (高难度) · AVX-512 + polyhedral · 3-6 月
**思路**: 多维循环嵌套 + AVX-512 zmm (8 路 f64)

**需要**: LLVM 级别的 polyhedral 分析 / tiling / fusion
**收益**: matvec 8x, 大规模矩阵 4-10x 接近 MKL/BLAS
**风险**: 极高, 工程量等同小型研究项目

## 五、为什么目前不直接做 L2

1. **需要类型系统更稳** — Phase 1 浮点列表推断目前有 workaround (`+0.0`), 真正的 SIMD 需要**可靠**的 `__浮点列表__` 标记, 不能基于 workaround
2. **依赖分析缺失** — Nova 当前无别名分析, 无法判断 `a[j] = b[j] + c[j]` 中 a, b, c 是否重叠
3. **寄存器分配简单** — Nova 目前用栈模型, 所有运算经过 rax/rcx, 没有真正的寄存器分配器
4. **ROI 评估** — 当前 Nova 浮点性能与 C -O0 相当, 做 L1 intrinsic 已能让 AI 算子 2-4x. L2 需要先解决上游类型+寄存器问题
