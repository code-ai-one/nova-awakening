## 二、Rust 种子的技术债 (✅ 已根治!)

### 2026-04-17 更新: Rust 2 patch 全部移除

**之前记忆中的 2 处 workaround patch 已源码根治**:
1. ✅ **`patch_stage2_dict_inline_strcmp`**: 
   - 根治路径: `前端入口_运行时种子.nova` **3 处内联 strcmp 改为 call 字符串相等**
   - Rust 种子中 `patch_runtime_blob_dict_inline_strcmp` 已删除 (linker.rs:1416 注释)
   
2. ✅ **`patch_elf_zero_fill`**: 
   - 根治路径: `模块_写LinuxELF` 使用 **Nova 原生 `零内存(N)` + `模块_分块写文件`**
   - Rust 种子中 `patch_elf_zero_fill` 已删除 (main.rs:699 注释)
   
3. ✅ **`patch_stage2_init`**: 函数废弃 (main.rs:700)
   - 本应修复 `__初始化全局__` 代码生成根因, 不靠二进制 NOP

**Fail-Fast 保护**: `NOVA_ENABLE_STAGE2_PATCH` 环境变量仍作为 fail-fast, **禁止复活** patch

**分离链式自举 (当前状态)**:
```
Stage-0 (Rust 种子, 纯引导器, 无 patch)
   ↓
Stage-1 (Nova, 由 Rust 种子编译 阶段0_种子编译器)
   ↓
Stage-2 == Stage-3 (byte-identical ✓, diff_bytes=0)
```

### 完全脱离 Rust 的真正要求 (剩余)
- ~~**条件 A**: 无内联 strcmp~~ ✅ 已根治
- ~~**条件 B**: Nova ELF 零初始化完整~~ ✅ 已根治  
- **条件 C**: 用 Nova 重写 Rust 种子 10000+ 行 → 完全无 Rust 依赖
  - 子步骤 C.1: 词法.nova (已做 7/7 自测, 独立分支)
  - 子步骤 C.2: 语法.nova (未做)
  - 子步骤 C.3: 后端.nova (未做, 最复杂)

**结论**: 当前 Nova **真正独立自举** (Stage-0 Rust 是合法的引导器, 不含 workaround). **"完全摆脱 Rust"** 只差 C.2/C.3 (Nova 重写词法/语法/后端), 这是**独立工程选择**, 不是技术债.

## 四、未解决 Bug 列表

### B1: ai_float_ops 30/32 (sigmoid + softmax)
- `sigmoid(2) > 0.8` 失败
- `softmax 和 = 1` 失败
- **待调查**: 数值精度还是 Nova 编译器 bug

### B2: matvec_bench iter3 (64+32 得 84 非 96)
- 独立测试 `c = r1[0] + r2[0]` 正确
- `sum12 + s3` (第三次加) 错
- 可能原因: 栈偏移/寄存器冲突/IEEE load 深层
- **待调查**: 反汇编 + 逐指令追踪

### B3: 变量名冲突无 parser 报错
- 或/与/非 作变量名 → 静默接受 → 运行错
- **待修**: parser 加保留字检查

### B4: Rust 种子 2 处 patch 未根治
- 见第二节

### B5: ai_float_ops 2 失败深因?
- 可能与浮点精度/算法设计有关, 不一定是编译器问题

## 七、此时的诚实定位

**当前 Nova** 是一个:
- 📚 **学术级研究编译器** (字节级自举证明领先)
- 🎓 **教学级示范** (清晰中文源码, 16KB 产物)
- 🧪 **实验级语言** (模糊逻辑/连续值等创新方向)

**而非**:
- ❌ 商业生产级
- ❌ 替代 rustc/gcc 的工具链
- ❌ 完全自主 (Rust 种子仍需)

**但**:
- ✅ 基础扎实, 可持续迭代
- ✅ 设计独特, 不抄袭
- ✅ 作者可控, 无外部依赖的发展阻力

**诚实评分** (相对完美商业级):
- 自举/正确性: 8/10 (高, 仍有 bug)
- 性能: 4/10 (fib 已 3x, 大规模仍慢)
- 工具链完整性: 3/10 (编译器+链接器, 无调试/包管理)
- 生态: 0/10 (单人项目)
- 创新性: 9/10 (中文/字节级自举/连续值)
- **总体**: 距完美商业级约 **40%**
