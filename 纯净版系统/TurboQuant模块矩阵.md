# TurboQuant 模块矩阵

## 0. 使用说明

本矩阵用于把主文档中的判断压缩成可执行清单。

字段说明：

- **当前能力**：模块现在已经具备的能力
- **主要缺口**：与 TurboQuant / PolarQuant / QJL 原生改造目标之间的距离
- **定位级别**：主改造 / 协同重定位 / 保留观察 / 背景旁证
- **改造关系**：它和其他模块之间的牵引关系

## 5. 关键改造依赖图

### 5.1 第一批必须收敛的关系

- `网络量化 -> 权重转换 -> 全连接层`
- `语义嵌入 -> 检索增强推理 -> 记忆管理`
- `分页注意 -> 快注意 -> 多头注意力`
- `工作记忆记 <-> 记忆管理`

### 5.2 第二批协同关系

- `在线学习 -> 网络量化 / 压缩校准`
- `云端桥接 -> 知识蒸馏 -> 本地压缩部署`
- `索引客户端引 -> 语义记忆持久化`

### 5.3 不应被误设为主依赖的关系

以下模块不应在架构图中被放到主链中心：

- `赫布学习更新`
- `策略梯度`
- `门控循环`
- `马尔链创建`
- 图形/渲染/图片压缩/页面控制链

## 6. 执行顺序建议

### 6.1 底层重写批次

- 批次A：`网络量化` / `权重转换` / `定点精度` / `全连接层`
  - 先统一量化表示、误差边界与权重布局，否则后续模块没有共同底座
- 批次B：`分页注意` / `快注意` / `多头注意力`
  - 再统一 KV 页块格式、分页统计与增量解码验证，否则注意力链仍只是桥接并轨
- 批次C：`语义嵌入` / `检索增强推理`
  - 再统一语义表示、压缩索引、分块策略与检索质量验证，否则 RAG 仍停留在工程可用层
- 批次D：`记忆管理` / `工作记忆记`
  - 再统一 STM / EM / LTM / SDM 的压缩记忆块格式，否则记忆层继续与 KV 层平行演化
- 批次E：`在线学习` / `知识蒸馏` / `云端桥接`
  - 最后接系统级校准、迁移和部署，否则压缩底座无法形成完整系统闭环

### 阶段一：补齐真实基线

- 统一量化抽象
- 扩展已落地的 `RAG` 真实相似度与排序，实现检索质量基准与异常回归检查
- 为 `语义嵌入`、`工作记忆`、`记忆管理` 建指标

### 阶段二：统一压缩记忆格式

- 设计 KV 页块格式
- 设计工作记忆块格式
- 设计语义记忆压缩索引格式

### 阶段三：系统级整合

- 云边迁移适配
- 在线学习校准接入
- 持久化与检索后端接入

## 7. 矩阵结论

真正的主改造路线不是“把所有带 AI、优化、压缩字样的模块都列入中心”，而是只围绕以下主链展开：

- `网络量化`
- `权重转换`
- `定点精度`
- `分页注意`
- `快注意`
- `语义嵌入`
- `检索增强推理`
- `记忆管理`
- `工作记忆记`

其余模块要么是协同层，要么是旁证层，要么只是背景系统。

## 8. 持续完善模板

如果后续继续扩大到 `1500+` 模块，建议所有新模块都按同一模板回写，避免矩阵越写越散。

### 8.1 新模块填表规则

- 每个模块只保留一行核心结论
- 当前能力只写已经能直接确认的内容
- 主要缺口只写与主链真正相关的距离
- 定位级别必须在四类中选一个，不能模糊
- 改造关系必须说明它和谁产生牵引或依赖

### 8.2 模块归类优先级

- 先判断它是否属于 `量化 -> 注意力/KV -> 语义嵌入 -> 检索 -> 记忆` 主链
- 再判断它是否是协同层，能否服务主链
- 再判断它是否只是背景工程能力
- 最后才决定是否保留为观察项

### 8.3 新增模块的最小记录单元

每个新增模块至少记录以下五项：

- 模块名
- 当前能力
- 主要缺口
- 定位级别
- 改造关系

### 8.4 矩阵维护原则

- 不因为“看起来智能”就升格为主改造
- 不因为“功能多”就默认属于核心链
- 不因为“已经能跑”就忽略缺口
- 不因为“暂时用不上”就删除旁证价值
