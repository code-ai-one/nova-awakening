## 用途
每次改动编译器/运行时/代码生成器后必跑，确认没有回归。

## 步骤

1. 清除Stage-1运行时blob缓存（确保从源码重新生成）
// turbo
```bash
rm -f /home/cch/桌面/新觉醒/分离链式自举/阶段1_自举启动/_stage1_build/stage1_runtime_blob.bin
```

2. 重新编译Rust种子编译器
// turbo
```bash
cd /home/cch/桌面/新觉醒/分离链式自举/种子/rust生成器 && cargo build --release 2>&1 | tail -5
```

3. 运行完整自举链并过滤关键输出
```bash
cd /home/cch/桌面/新觉醒/分离链式自举/阶段0_种子编译器 && /home/cch/桌面/新觉醒/分离链式自举/种子/rust生成器/target/release/nova_kernel_aux 2>&1 | grep -E "dict-strcmp|dict-patch|zero-fill|diff_bytes|fixed_point|convergence_round"
```

4. 期望输出
- `fixed_point: true`
- `diff_bytes=0 size2=<N> size3=<N>`（两者相等）
- `convergence_rounds=2`（可能1-3之间）

5. 二进制等价性强验证
// turbo
```bash
cmp /home/cch/桌面/新觉醒/分离链式自举/阶段1_自举启动/阶段2_编译器 /home/cch/桌面/新觉醒/分离链式自举/阶段1_自举启动/阶段3_编译器 && echo "✓ Stage-2==Stage-3"
```

6. 如果失败
- 定位diff_bytes在哪个区域（代码段/数据段/gap）
- 追溯到具体Nova源文件或Rust种子代码
- 修复根因后重跑，直到通过
- 写入MEMORY.md第七节
