### ⚠️ `sigmoid(2) > 0.8` 失败 — 算法问题
```
bits = 4604930618986332159 ≈ 0.76 
期望 0.881 > 0.8
实际 0.76 < 0.8 ✗
```
**根因**: Taylor 4 阶展开对 `exp(-2)` 精度不足:
- 真实 exp(-2) ≈ 0.1353
- Taylor 4: 1 - 2 + 2 - 1.333 + 0.667 = 0.333
- 误差 0.198, sigmoid 结果偏差 → 不满足 `> 0.8`
- **这是测试算法设计问题, 非编译器 bug**

### ⚠️ `softmax 和 = 1.0` 精度边缘
```
理论 r0 + r1 = (ea + eb) / s = s/s = 1.0
但 IEEE 浮点除法 round-to-nearest, 可能精度 ±ε
tolerance 0.001 边缘, 偶尔失败
```
**非编译器 bug, 是数值分析容差问题**
