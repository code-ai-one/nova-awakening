## 二、正确方案: Reproducible Bootstrap

### 核心哲学
```
永久种子 = 上次 Stage-3 的 binary (byte-identical 可重现)
```

### 工作流
```bash
# 第一次 (一次性, 已完成):
Rust 种子 → Stage-1 → Stage-2 → Stage-3 ✓ byte-identical
保存 Stage-3 binary 作为 seed_0.bin

# 之后每次编译 Nova 自身:
seed_0.bin → 编译阶段0_种子编译器 → Stage-1 → Stage-2 == Stage-3
验证: Stage-3 == seed_0.bin ?
若是: 使用 Stage-3 作为新 seed (可选更新)
若否: 检查改动是否有意, 修复或回滚
```

### 对标工业先例
| 语言 | 引导方式 |
|---|---|
| **GHC (Haskell)** | checked-in "GHC.hs" bootstrap binary |
| **Guile** | checked-in pre-built elisp bytecode |
| **SBCL** | self-hosted, checked-in image |
| **Rust** (实际上) | bootstrap from previous rustc binary |
| **Nova** (本方案) | **checked-in Stage-3 binary (7.5MB)** |

### 优势
- ✅ **彻底摆脱 Rust 工具链**
- ✅ Nova 可在任何 Linux x86-64 上从头编译自己
- ✅ 字节级自举已证明可重现 (diff_bytes=0)
- ✅ 单一源码真相 (517 模块 + 1 binary)
- ✅ 不需要月级工程重写 backend_seed.rs

### 实施步骤
1. **保存当前 Stage-3** binary 为 `/分离链式自举/seed_stage3.bin`
2. **创建 shell 脚本** `bootstrap_from_stage3.sh`:
   ```
   seed_stage3.bin 编译 阶段0_种子编译器/ → new_stage1
   new_stage1 编译 原生编译器/ → new_stage2
   new_stage2 编译 原生编译器/ → new_stage3
   diff new_stage2 new_stage3 → expect 0
   ```
3. **废弃 Rust 种子** (或标记为 legacy, 仅保留作为 fallback)
4. **CI 验证**: 每次 commit 运行此脚本
